# UL1 — Clustering

**Paired reading:** `em-intuitive-explanation.pdf`, `em.pdf`, `nips15.pdf` (Kleinberg) · **Unit:** Unsupervised Learning

## The setting

No labels — only inputs. **Clustering** partitions the data into groups so that
points in a group are similar and points in different groups are dissimilar. The
hard part is that "similar," "group," and "correct" are all under-defined: with
no labels there is **no ground-truth error** to minimize. Everything in this
lecture is shaped by that fact.

## k-Means

The workhorse. Pick `k`, then alternate:

1. **Assignment step:** assign each point to its nearest centroid.
2. **Update step:** move each centroid to the mean of its assigned points.

Repeat until assignments stop changing. This is **coordinate descent** on the
within-cluster sum of squared distances (the "distortion"), so it **converges**,
but only to a **local optimum** — the result depends on initialization.

Properties to know:

- **Guaranteed to converge**, never guaranteed to converge *well*. Run it many
  times with different seeds and keep the lowest distortion; `k-means++`
  initialization spreads the initial centroids out and helps a lot.
- **Hard assignment** — each point belongs to exactly one cluster.
- **Implicit assumptions:** clusters are roughly **spherical**, similarly sized,
  and similarly dense. It uses Euclidean distance, so **feature scaling is
  mandatory** (same trap as kNN in SL4).
- **Choosing `k`:** there is no labels-based answer. Use the **elbow method** on
  distortion, the **silhouette score**, or domain knowledge — all heuristics.

## Soft clustering and the EM algorithm

k-means commits hard. **Expectation-Maximization** for a **Gaussian Mixture
Model** is the soft, probabilistic generalization — and it is OL4/OL5's Bayesian
machinery applied to a **latent variable** (the unobserved cluster label).

Model: the data is generated by `k` Gaussians, each with its own mean,
covariance, and mixing weight. The cluster label is the hidden variable. EM
alternates:

- **E-step:** given current parameters, compute each point's **responsibilities**
  — the posterior probability it belongs to each Gaussian. (This is Bayes' rule
  from OL5; the cluster is the hypothesis.)
- **M-step:** given those soft responsibilities, re-estimate each Gaussian's
  parameters by responsibility-weighted maximum likelihood.

Properties:

- EM **monotonically increases the data likelihood** each iteration and
  **converges to a local optimum** — same init-sensitivity story as k-means.
  (k-means is essentially EM with hard assignments and fixed spherical
  covariances.)
- **Soft assignment** — a point can be 70% cluster A, 30% cluster B. This handles
  overlapping clusters and, via full covariance matrices, **elliptical** clusters
  that break k-means.
- More flexible ⇒ more parameters ⇒ more data needed and more ways to overfit
  (e.g. a Gaussian collapsing onto a single point, likelihood → ∞).

## Hierarchical clustering

Builds a tree (**dendrogram**) instead of a flat partition. **Agglomerative**:
start with every point its own cluster, repeatedly merge the two closest
clusters. The **linkage** defines "closest": single (nearest pair — produces
chaining), complete (farthest pair — compact clusters), average, or Ward
(minimize variance increase). Advantage: no need to pick `k` up front — cut the
dendrogram at any level. Cost: `O(n²)` or worse, and merges are greedy and never
undone.

## Density-based clustering

**DBSCAN** grows clusters from dense regions: points with enough neighbors within
radius `ε` are "core" points; clusters are connected core regions; everything
else is **noise**. It finds **arbitrarily shaped** clusters (not just blobs),
**doesn't need `k`**, and labels outliers explicitly — but it struggles when
clusters have very different densities and is sensitive to `ε`.

## Evaluating clusters without labels

The recurring problem. Tools:

- **Internal** (no labels): **silhouette score** (cohesion vs. separation per
  point), within-cluster sum of squares / distortion, the elbow method,
  Davies–Bouldin index. For GMM/EM, **BIC/AIC** trade likelihood against
  parameter count to pick `k`.
- **External** (if you happen to have labels for validation): adjusted Rand
  index, normalized mutual information.
- **Visualization** — project to 2-D (PCA, t-SNE from UL3/UL4) and look.

The UL report lives on these — with no ground truth, your figures and internal
metrics *are* the evidence.

## Kleinberg's Impossibility Theorem

The conceptual climax (`nips15.pdf`), and a final-exam favorite. Kleinberg
proposed three properties any "reasonable" clustering function should satisfy:

- **Scale invariance** — scaling all distances by a constant doesn't change the
  clustering.
- **Richness** — every possible partition is achievable by *some* distance
  function.
- **Consistency** — shrinking within-cluster distances and stretching
  between-cluster distances doesn't change the clustering.

**The theorem: no clustering function can satisfy all three simultaneously.**
The takeaway is not "clustering is hopeless" — it's that clustering inherently
involves **trading off desirable properties**; every algorithm gives up at least
one, and "the correct clustering" is not a well-defined notion independent of
your goals. It is the unsupervised echo of No Free Lunch (OL3): no universal
clusterer, just biases matched to purposes.

## Review Questions

**Conceptual checks**

1. Why is there no "training error" to minimize in clustering, and how does that
   shape the entire problem of evaluation?
2. State k-means' implicit assumptions about cluster shape, size, and density.
   Sketch a dataset where each assumption is violated.
3. Hard vs. soft assignment: what does each mean, and which algorithm gives
   which?

**Derivation / math**

4. k-means is coordinate descent on distortion. Identify the two coordinate
   blocks and explain why alternating them guarantees convergence — and why only
   to a local optimum.
5. In EM for a GMM, what does the E-step compute and via which rule from OL5?
   What does the M-step compute? Why does the data likelihood never decrease?
6. Argue that k-means is a special case of EM. What two restrictions on the GMM
   recover k-means?

**Analysis — "why does it behave this way"**

7. You run k-means twice with different seeds and get different clusters.
   Explain why, and give two ways to get a trustworthy result.
8. EM with full covariance matrices can fit elliptical, overlapping clusters
   that k-means cannot. Explain what extra modeling capacity buys that — and
   name a way EM can pathologically overfit.
9. Why does DBSCAN find shapes that k-means cannot, and where does DBSCAN itself
   break down?

**Exam-style**

10. State Kleinberg's three properties and the impossibility result. What is the
    correct *interpretation* — and how is it analogous to No Free Lunch?
11. You must pick `k` with no labels available. List three principled methods and
    say what each actually measures.
12. For the UL report you must justify a clustering with no ground truth.
    Describe the specific metrics and figures you'd present and what claim each
    one supports.
